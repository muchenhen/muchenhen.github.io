对每组测试用例，在一行中输出”Case #X: true”如果A+B>C，否则输出”Case #X: false”，其中X是测试用例的编号（从1开始）。

2147483647 0 2147483646

0 -2147483648 -2147483647

输出例子**:**

Case #1: false

Case #2: true

Case #3: true

Case #4: false

LeetCode题解7.反转整数 2018-08-30|C++

反转整数

给定一个 32 位有符号整数，将整数中的数字进行反转。

示例 1:

输入**:** 123

输出**:** 321

示例 2:

输入**:** -123

输出**:** -321

示例 3:

输入**:** 120

输出**:** 21

注意**:**

假设我们的环境只能存储 32 位有符号整数，其数值范围是 [−231,  231 − 1]。根据这个假设，如果反转后的整数溢出，则返回 0。

Given a 32-bit signed integer, reverse digits of an integer.

Example 1:

Input: 123

Output: 321

Example 2:

Input: -123

Output: -321

Example 3:

Input: 120

Output: 21

Note:
Assume we are dealing with an environment which could only store integers within the 32-bit signed integer range: [−231,  231 − 1]. For the purpose of this problem, assume that your function returns 0 when the reversed integer overflows.

分析：

假设原数字为a，我们的目标数字是b，则要想办法求得a的最后一位数字，并作为b的第一位数字。求最后一位数字可以用a%10得余数，之后/10则可以继续求下一位数字。

//pop operation:

pop = x % 10;

x /= 10;

//push operation:

temp = rev * 10 + pop;

rev = temp;

注意事项为溢出。简单做这种操作容易造成溢出。可以进行事先的检查来防止溢出，

复杂度分析

  • 时间复杂度：O (log(x)) O(log(x))，x 中大约有 log​10​​(x) 位数字。

  • 空间复杂度： O(1)。

指定一个基准数，一般指定数列第一个数，之后从数列两端开始分别寻找比基准数大和小的数，然后进行交换位置。比起冒泡排序，快速排序的比较和交换是跳跃式的。

例如对以下数列进行快速排序：6、1、2、7、9、3、4、5、10、8

指定6为基准数，设定两个变量i、j，i从左至右寻找小于6的数字，从右往左寻找大于六的数字。第一步j=8发现大于6，i=1小于6，不交换，继续寻找，j=10，i=2，继续寻找，j=5小于6，i=7大于6，交换i、j的位置，此时数列变为6、1、2、5、9、3、4、7、10、8，继续寻找交换9与4，数列变为6、1、2、5、4、3、9、7、10、8，此时i与j都寻找到3，比较发现3小于6，则3应该交换到左侧，将6换至中间，数列变为3、1、2、5、4、6、9、7、10、8。在对6左右两侧重复之前操作。

冒泡排序

  1. 简要描述

    冒泡排序每次比较两个相邻元素，若顺序错误就交换位置。

    如对数列9、6、2、5、4、8经行从小到大排序，从9开始，9>6，交换位置变为6、9、2、5、4、8，重复直到6、2、5、4、8、9。第一趟排序结束。然后再对6进行排序，进行第二趟，直到所有数字排序完毕。最后一个数字则不需要进行，并且每一趟所需要比较的次数都比上一次少。
